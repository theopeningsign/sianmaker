> 관련 데모: `demo-sign.html` (CSS로 만든 한글 채널/네온/금속 렌더 근사치)

## Context — 왜 이 논의를 했나

집에서 재미나이와 나눈 "LoRA 회의론"을 검토하다가, 한글 간판 생성의 근본 전략을 재설계하게 됨.
핵심 깨달음: **AI는 한글 글자를 "더 잘" 만들어준 적이 없다. 시키면 깨진 한글이 나온다.** 따라서 글자 생성에서 AI를 빼는 건 품질 손해가 아니라 깨짐 0% 달성.

## 확정된 전략 결정

1. **로컬 LoRA 학습 전면 취소** — Flux Pro는 폐쇄형 API라 로컬 `.safetensors` 업로드 불가. OMEN 16(8GB)으로 Flux 학습은 ROI 최악. → OMEN은 개발/코딩 PC로만.
2. **한글 글자는 AI로 절대 만들지 않는다.** 진짜 폰트로 렌더 → 깨짐 0%.
3. **AI는 글자가 아니라 "사진 마감 · 건물 합성"에만 쓴다.**

## 핵심 아키텍처 — 절차적 렌더 + AI 사진 마감 (트리플 하이브리드)

```
[Stage 1] 절차적 렌더 (WebGL / Three.js) — 비용 0원, 즉시, 한글 100% 정확
  · 간판 구조(3D 지오메트리) + 폰트 글자 + 발광/입체/재질을 코드로 렌더
  · step3에서 받은 정확한 가로×세로 비율을 그대로 보존 (AI 풀생성은 못 하는 강점)
  · 유저가 색·폰트·글자 바꿔도 실시간 0원 반영

[Stage 2] (옵션) AI 로고 입체화 — 약 40원
  · 유저가 올린 평면 로고(.png) → img2img로 금속/아크릴 입체 오브젝트화
  · 상호명만 있고 로고 없는 경우 많음 → 핵심 아님, 부가기능

[Stage 3] AI 사진 마감 패스 — 약 50~70원 (최종 "시안 생성" 시 1회)
  · 절차적 렌더 결과를 "구조 잠금장치(앵커)"로 사용:
      - 렌더에서 뽑은 Canny(윤곽)/Depth(깊이)를 ControlNet으로 하드 락
      - img2img 저강도(denoise 0.2~0.4)
  · AI는 글자를 못 움직임(구조 고정) → 한글 100% 유지
  · AI가 하는 일: 사진 같은 재질·그레인·실제 광택 + 건물 사진에 합성(접촉그림자·벽 반사광·톤 일치)
  · "그림 같던 것 → 실제 시공 사진"으로 넘기는 단계. 간판은 잠그고 환경만.

→ 모든 결과물은 투명 PNG → 기존 Fabric.js 캔버스(드래그/스케일/회전) 그대로 호환
```

**라이선스 주의:** ControlNet은 Flux **Dev** 계열에서 돈다(Pro 아님). Flux Dev는 비상업 라이선스이므로 **반드시 Fal.ai/Replicate 등 상업계약된 호스팅 API로만** 사용. 자체 호스팅 금지.

## 검증된 것 / 미검증 리스크

- ✅ 검증됨: CSS 데모(`demo-sign.html`)로 네온/채널/금속 한글 렌더 → 한글 안 깨지고 그럴듯함. 특히 네온 우수.
- ⚠️ 사장님 피드백: "아직 그림 같다, 실제 사진 느낌은 아니다" → 맞는 지적. 절차적만으론 CG 티.
- 🔴 **미검증 1순위 리스크:** Stage 3 마감 패스(저강도+ControlNet)가 **한글 안 깨고 사진급으로** 올려주는가? → API 붙이면 가장 먼저 검증할 것.
- 그림→사진 격차 닫는 두 레버: ① WebGL PBR+사진 post(0원, 격차 70~80%) ② AI 마감 패스(나머지 + 건물 융합).

## 5종 간판 적용 가능성

공식 = **[파라메트릭 3D 구조물] + [폰트 글자] + [AI 사진 마감]**

| 간판 | 구조 | 판정 |
|---|---|---|
| 전면간판 | 벽면 평면/박스 | ✅ 핵심, 가장 잘됨 |
| 돌출간판 | 튀어나온 박스+브래킷 (WebGL 강한 영역) | ✅ 됨 |
| 어닝간판 | 경사 천막 | ✅ 됨, 단 천 재질은 AI 마감 의존 |
| 실내간판 | 전면과 동일, 실내사진 합성 | ✅ 됨 |
| 지주형간판 | 기둥+상단패널, 땅에 세움 | ✅ 됨, 바닥 접촉그림자 AI 마감 |

**안 되는 것 (스코프 제외 / 별도 처리):**
- **"기타 / 기타조형물"** = 임의 자유 조형물 → 파라메트릭 불가. v1 미지원으로 스코프 긋기.
- **임의 업로드 로고 입체화** → Stage 2(AI img2img) 필요.
- **플렉스/지주 패널의 복잡 그래픽 디자인** → 렌더 문제 아닌 디자인 문제(별도 편집/업로드).
- **천(어닝/패브릭) 사실감** → AI 마감 품질 의존도 높음.

**스코프 현실:** 카테고리마다 전용 파라메트릭 3D 템플릿 필요 = 5개 지오메트리 빌더 + 재질 변형 = 상당한 분량.
→ **빌드 순서: 전면(채널/플렉스) 먼저 끝장 → 돌출 → 어닝 → 지주 → 실내.** 한 번에 다 만들지 말 것.

## 프롬프트 전략의 변화 (중요)

이 방향에서는 **거대한 영문 프롬프트 라이브러리가 불필요**해진다. 프롬프트 역할이 엔진→핸들로 축소.

| | 옛날(AI 풀생성) | 지금(절차적+마감) |
|---|---|---|
| 정밀작업 대상 | 영문 프롬프트 라이브러리(조합별 수백개) | **렌더 파라미터 프리셋(JSON: 템플릿·재질·폰트·색·비율)** |
| AI 프롬프트 | 길고 복잡, 핵심 | 짧고 재사용, 보조 (재질군×주야 10~20개) |
| 결제후 LLM 프롬프트 확장 | 필수 | **거의 불필요** (선택→파라미터 결정론적) → 비용·실패점 제거 |

즉 step1~3 유저 선택 → "프롬프트"가 아니라 **"렌더 프리셋"으로 매핑.** 정밀함은 사라지지 않고 형태만 바뀜.
(주의: 만약 AI 풀생성으로 회귀하면 거대 프롬프트 라이브러리 다시 필요.)

## 텍스트+일러스트 통합 채널 (예: "대박 🐔 치킨")

글자와 그림(마스코트)이 한 몸인 통일된 채널간판도 절차적으로 가능 — **오히려 강점.**
- 그림이 **벡터 패스**면 글자 글리프와 똑같은 파이프라인(압출+면발광+bloom+재질)을 타서 재질 100% 일치. (AI 풀생성은 글자/그림 재질 불일치·한글깨짐으로 이걸 잘 못함)
- 그림 출처: ① **업종별 클립아트 라이브러리(닭·커피잔·가위…)** = v1 추천, 완벽 품질 / ② 단순·단색 로고 자동 벡터화(potrace) / ③ 복잡·사진 로고 = 채널화 불가 → 평면 인쇄 or AI 별도.
- 레이아웃("대박/그림/치킨")은 편집기에서 인라인 요소로 배치.

## 유저 두 모드 (탐색 / 제작) — MVP는 **제작 모드 우선**

유저 의도가 둘로 갈린다:

| | **제작 모드** (정한 사람) | **탐색 모드** (모르는 사람) |
|---|---|---|
| 의도 | "이 간판을 정확히 만들고 싶다" | "내 가게엔 뭐가 어울리지?" |
| 시작 | 종류 선택(step1~) | 업종 + 건물 사진 |
| 원함 | 정확한 치수·한글·편집 | 영감·컨셉 |
| 엔진 | 절차적 렌더 + AI 마감 | **AI 풀생성** (여기선 약점 무관, 강점=즉석 다양성) |

- **MVP = 제작 모드 완성 우선.** 근거: 가격비교 플랫폼 유입 = 이미 어느 정도 알아본 고전환 유저.
- 탐색 모드 = 후순위 깔때기. 미정 유저를 AI 컨셉 시안으로 유입 → 방향 정해지면 제작 모드로 인계.
- **반전:** 접어뒀던 "AI 풀생성"은 죽은 게 아니라 **탐색 모드 전용으로 자리 이동.**
- 탐색 추천 로직은 AI 불필요 — 업종→간판종류 매핑 규칙표면 충분. AI는 비주얼 컨셉 시안에만.

## 제작 모드 완성 — 남은 경로 (MVP 크리티컬 패스)

## 전면-채널 시공방법 표현 = "백킹 레이어 프리셋"

시공방법은 **글자를 바꾸지 않는다.** 글자 레이어는 고정, **글자 뒤 배경/장착(백킹)만 교체.**
(step2에 레퍼런스 사진 보유: `맨벽/전면프레임/프레임바/파사드.JPG` → 이에 맞춰 렌더)

| 시공 | 모습 | 렌더 | 난이도 |
|---|---|---|---|
| **맨벽** | 글자를 벽에 직접 부착(스페이서), 글자 사이로 실제 벽 보임 | 백킹 없음 → 글자만 투명PNG, AI가 띄움그림자+벽 빛번짐 | 🟢 가장 쉬움·자연스러움 |
| **전면프레임** | 전면 사각 백판 위에 글자, 전체가 하나의 판 | 글자 뒤 사각패널(색·재질) + 테두리 절차적 추가 | 🟢 쉬움 |
| **프레임바** | 풀패널 대신 바/레일 구조에 장착 | 글자 뒤 바/레일 구조 절차적 추가 | 🟡 중간 |
| **파사드** | 외관을 건축적으로 디자인, 간판이 그 일부 | 거의 외관 재구성 → AI 마감 의존 큼 or 템플릿 | 🔴 가장 어려움 |

- **프로토타입 순서: 맨벽 → 전면프레임 → 프레임바 → (파사드 후순위).** 맨벽이 "글자렌더+건물합성" 핵심만 깨끗이 검증.
- **합성 주의:** 맨벽은 벽 위에 글자만. 전면프레임/파사드는 기존 외관 일부를 **덮으므로** 캔버스에서 마스킹 필요.

## 비즈니스

- 원가: 평면 ~30~40원, 입체 마감 ~50~70원 (+LLM 거의 0, RMBG, 재생성 여유) → 1,000원 결제, 마진 90%대 방어.
- 절차적 렌더 단계가 0원이라 유저는 디자인을 무한히 만지고, AI 비용은 최종 생성 1회에만 집중.
